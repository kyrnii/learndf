# 地图系统设计

地图相关层统一放在 `src/world/layers` 下，当前分为两类：
- 真相层：`TerrainMap`、`OccupancyMap`
- 派生层：`NavigationMap`、`NavMeshLayer`、`StaticPhysicsLayer`

## 真相层
`TerrainMap`

- 保存地图边界和格子尺寸
- 决定天然不可走、不可放区域

`OccupancyMap`

- 保存放置物和建筑 footprint
- 是放置合法性的主真相源
- 支持序列化和反序列化

## 派生层
`NavigationMap`

- 基于 `TerrainMap + OccupancyMap` 判断可走性
- 提供直线可达检测
- 提供简单 detour
- 提供网格 A* 路径搜索
- 支持三种离散体型：`small`、`medium`、`large`

`NavMeshLayer`

- 只针对轴对齐地形和矩形占用
- 把连续可走格子合并成矩形区域
- 为区域建立邻接关系
- 按 chunk 管理区域缓存
- portal 使用边段而不是单点
- 路径会先做 portal string-pulling，再做最小 line-of-sight 平滑
- 按体型分别缓存导航结果
- 地图变化时只标脏，不立即重建
- 只有查询某个体型时，才懒失效并重建该体型缓存
- 已存在的体型缓存只重建受影响的脏 chunk，而不是整档全量重建

`StaticPhysicsLayer`

- 基于 `OccupancyMap` 生成静态物理障碍
- 支持整层开关
- 支持延迟刷新
- 支持相邻占用合并成更大的矩形刚体

## 当前放置流程
1. `PlacementSystem` 检查能否放置
2. 写入 `OccupancyMap`
3. 按需更新地形阻挡
4. 标记 `NavigationMap / NavMeshLayer / StaticPhysicsLayer` 脏区
5. `World.update(dt)` 只刷新运行时层

说明：
- `StaticPhysicsLayer` 会在 `World.update(dt)` 中刷新
- `NavMeshLayer` 不在 `World.update(dt)` 中重建，而是在查询时懒刷新
- 放置和地形变更会把 footprint 对应的 chunk 标脏，供 navmesh 查询时局部重建

## 当前寻路策略
`MapContext.findPath(...)` 按这条顺序工作：

1. 优先走 `NavMeshLayer`
2. 如果 navmesh 不可用或失败，退回 `NavigationMap`
3. `Locomotor` 继续消费统一的 waypoint 结果

## 体型寻路
当前只保留三档离散体型：

- `small`：`clearanceCells = 0`
- `medium`：`clearanceCells = 1`
- `large`：`clearanceCells = 2`

含义是同一张地图上，不同体型会看到不同的“可通行空间”：

- `small` 更容易穿过狭窄通道
- `large` 会离障碍留出更多余量

## 后续建议
下一步最值得补的是：

1. 进一步完善 funnel 和更精细的路径观感调整
2. navmesh 的 portal 级平滑与更细粒度的局部图更新
3. 放置失败原因，例如越界、地形不允许、占用冲突
