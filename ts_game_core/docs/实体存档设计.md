# 实体存档设计

当前实体存档主链已经建立在 `prefab` 系统之上。

## 核心原则

- `prefab` 负责“这个实体怎么生成出来”
- `component.serialize()` 负责“这个组件当前运行数据怎么保存”
- 恢复时先 `spawnPrefab(...)`
- 再对组件调用 `deserialize(...)`

## 当前入口

### 保存

- `entity.serialize()`

返回的数据包含：

- `prefabId`
- `prefabName`
- `mapId`
- `tags`
- `components`

### 恢复

- `restoreEntityFromSave(registry, save, options)`

对应文件：

- `src/prefabs/restoreEntity.ts`

## 为什么要先 prefab 再 deserialize

因为 prefab 才知道：

- 要加哪些组件
- 默认的 SG / Brain / Physics 配置
- 默认的 Placeable 配置

如果没有先按 prefab 生成出实体，组件实例都不存在，就没法正确反序列化。

所以正确顺序一定是：

1. 按 `prefabId` 生成实体
2. 让 prefab 先完成默认装配
3. 再把存档里的组件数据回填进去

## 当前已支持序列化的组件

- `Transform`
- `Health`
- `Inventory`
- `Item`
- `Placeable`
