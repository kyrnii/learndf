# 放置系统设计

`PlacementSystem` 是地图层上的独立系统，负责处理放置预览、占用写入和地形阻挡更新。

对应代码：

- `src/world/PlacementSystem.ts`
- `src/components/Placeable.ts`

## 当前职责

`PlacementSystem` 负责：

- 放置预览
- 放置合法性校验
- footprint 旋转
- 写入 `OccupancyMap`
- 更新地形阻挡
- 标记地图派生层脏区

它不再直接创建物理障碍。

静态物理障碍现在由 `MapContext.staticPhysics` 根据当前占用统一刷新。

## Placeable 组件

`Placeable` 会在实体进入世界、切换地图、离开世界时自动调用放置系统。

当前保留的核心字段有：

- `placementId`
- `footprint`
- `rotation`
- `createPhysicsObstacle`
- `blockTerrainCell`
- `autoPlaceOnWorldJoin`
- `placed`

这里的 `createPhysicsObstacle` 不再表示“立即创建刚体”，而是表示“这块占用是否应参与静态物理层生成”。

## 预览结果

`preview(options)` 会返回：

- 当前地图实例
- 吸附后的格子坐标 `cellX / cellZ`
- 吸附后的世界坐标 `snappedX / snappedZ`
- 中心点 `centerX / centerZ`
- 旋转后的 footprint
- 当前是否可放 `canPlace`

## 放置流程

1. 解析地图实例
2. 计算吸附格子
3. 根据旋转得到 footprint
4. 用 `OccupancyMap.canPlaceRect(...)` 校验
5. 写入占用
6. 按 footprint 更新地形阻挡
7. 标记地图派生层脏区

## 与 Physics 的关系

放置合法性不依赖 physics。

physics 在这里的定位是：

- 可选增强层
- 负责真实碰撞表现
- 从地图占用派生静态障碍

也就是说：

- 地形层和占用层是真相源
- 静态物理层是派生结果

## 后续建议

下一步最值得补的是：

1. 放置失败原因，例如越界、地形不允许、占用冲突
2. footprint 掩码支持，让非矩形占用也能表达
3. 静态物理层的 chunk 化刷新，避免大地图全层重建
