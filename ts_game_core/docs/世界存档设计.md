# 世界存档设计

当前 `World` 已经支持整体导出为纯数据，并通过 prefab 系统恢复。

## 当前入口

### 保存

- `world.serialize()`

返回的数据包含：

- `defaultMapId`
- `maps`
- `brainTime`
- `sgTime`
- `entities`

### 恢复

- `restoreWorldFromSave(registry, save, options)`

对应代码：

- `src/world/restoreWorld.ts`

## 当前恢复顺序

1. 先根据存档创建地图实例
2. 创建 `World`
3. 恢复 world 级时间
4. 遍历实体存档
5. 每个实体先按 `prefabId` 生成
6. 再调用 `entity.deserialize(...)`

## 当前地图保存内容

每张地图当前会保存：

- `id`
- `terrain.cellSize`
- `terrain.bounds`
- `terrain.blockedCells`

占用图和导航图目前不单独保存：

- `OccupancyMap` 会在放置类实体恢复时重新建立
- `NavigationMap` 会根据地形和占用重新派生

## 当前限制

- 物理世界本身不直接序列化
- `Brain` 运行中的临时状态不单独保存
- `StateGraph` 当前状态不单独保存

这意味着当前存档更偏向“世界重建型存档”，而不是“逐帧精确热恢复”。

## 后续建议

下一步最值得补的是：

1. `Combat`、`Locomotor` 等组件存档
2. `StateGraph` 当前状态与时间轴进度存档
3. `Brain` 关键黑板/运行态数据存档
